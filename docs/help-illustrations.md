# Асуултын тусламжийн хөдөлгөөнт дүрслэлүүд — архитектур ба заавар

Махбодын сорилын 12 асуулт бүр «Хэрхэн тодорхойлох вэ?» товчтой. Товч нь
хөдөлгөөнт SVG дүрслэлтэй тусламжийн цонх нээж, тухайн шинжийг бодитоор шалгах
аргыг (жишээ нь бугуйн сорил) гурван үр дүнгийн төлөвтэй нь үзүүлнэ. Мөн
дэлгэцийн баруун талын туслах самбарт (desktop, `xl↑`) идэвхтэй асуултын
дүрслэл шууд харагдана.

## Файлын бүтэц

| Файл | Үүрэг |
|---|---|
| `src/lib/questionHelp.ts` | Асуулт бүрийн тусламжийн агуулга: сорилын нэр, заавар, Х/Ш/Б үр дүнгийн тайлбар, aria-label. Монгол хэлээр. |
| `src/lib/doshaUi.ts` | `DOSHA_COLOR` / `DOSHA_BG` / `DOSHA_LETTER` — асуултын карт, цонх, самбар гурав хуваалцдаг. |

## Дүрслэлийн гэрээ (заавал мөрдөнө)

- **Зай:** `viewBox="0 0 240 160"`. Доод зүүн булан (x 8–175, y 128–158) нь
  фазын тэмдэгт (`PhaseBadge`) зориулагдсан — зураг оруулахгүй.
- **Зураас:** үндсэн silhouette `stroke="currentColor"` w=2.5; хоёрдогч
  нарийвчлал w=2 (opacity .4–.6); хамгийн нарийн w=1.8. Бүгд round cap/join.
- **Өнгө:** фазын элементүүд `ACCENT.H/S/B` — CSS хувьсагч + hex fallback
  (`var(--primary, #00626b)` г.м.). Fallback нь дүрслэлийг бие даасан HTML
  хуудсанд (artifact, стайлгүй орчин) апптай яг ижил харагдуулна. SVG дотор
  Tailwind class хэрэглэхгүй.
- **9 секундын мастер цаг:** `phaseCss(p)` нь гурван фазын харагдалтыг
  удирдана — Х: 0–30%, Ш: 33–63%, Б: 66–96% (хоорондоо зөөлөн шилжилттэй,
  фаз бүр 4px өргөгдөж орж ирнэ).
- **Хореограф:** фаз доторх «үйл явдал» нь мөн 9с үргэлжлэлтэй keyframe-ээр
  тухайн фазын цонхонд байрлана (жишээ: `Q01`-ийн хуруудын ойртолт,
  `Q08`-ийн санах ойн зурвасын дүүрэлт). **Эцсийн байрлалыг атрибутад, зөвхөн
  офсетыг keyframe-д** бич — ингэснээр царцаасан төлөв гүйцсэн дүр зургийг
  үзүүлнэ.
- **Нэрийн угтвар:** class/keyframe бүр `qN-` угтвартай (q1–q12) — нэг хуудсан
  дээр олон дүрслэл зэрэг байж болно (цонх + самбар + галерей).
- **Хөдөлгөөнт элемент бүр `qN-anim` class авна.** Үүгээр:
  - `prefers-reduced-motion: reduce` үед бүх хөдөлгөөн зогсож, Х төлөв
    статикаар харагдана (үр дүн дээр дарж бусад төлөвийг харж болно);
  - `variant` (царцаалт) өгөгдсөн үед бүх хөдөлгөөн эцсийн байрлалдаа зогсоно.
- **Хязгаар:** transform/opacity-оос өөр шинж чанар бүү animate хий;
  эргэлт/масштабд `transform-box` + `transform-origin` заа.

## Шинэ буюу засварлах дүрслэлийн ажлын урсгал

1. Агуулгыг `questionHelp.ts`-д шинэчил (тест бүрэн байдлыг шалгана).
2. `QXX*.tsx`-г засварла — `Q01BoneWrist.tsx` нь лавлах хэрэгжүүлэлт
   (томруулагч-инсет хэв маяг, хореограф, cue-ний хоцролт).
3. Нүдээр шалга (repo root-оос):

   ```bash
   # нэг асуултын 4 төлөвийг (авто + 3 царцаасан) том хэмжээгээр рендерлэх
   npx tsx qa-one.ts <id> /tmp/qN.html   # qa-one.ts скриптийг түр үүсгэнэ,
                                         # доорх хэсгээс эх кодыг нь харна уу
   ```

   Хөдөлгөөнийг шалгахдаа Playwright-аар тодорхой хугацааны агшнуудад
   screenshot ав (жишээ нь 0.4с / 3.4с / 4.1с / 7.2с — фаз бүрийн үйл
   явдлын өмнө ба дараа).

4. `npx tsc --noEmit`, `npm test`, `npm run build`.

> `qa-one.ts` ба галерейн рендер скриптүүд repo-д commit хийгддэггүй түр
> хэрэгслүүд: `react-dom/server`-ийн `renderToStaticMarkup`-аар компонентыг
> рендерлээд HTML хуудсанд буулгаж, headless Chromium-аар screenshot авдаг.
> Хэрэгтэй үед `HELP_ILLUSTRATIONS`-оос импортолж дахин бичихэд ~40 мөр.

## Бие даасан хуудсанд зөөх (portability)

Дүрслэл бүр `<style>`-аа дотроо авч явдаг тул рендерлэгдсэн SVG markup-ийг
шууд хуулж ямар ч HTML хуудсанд ажиллуулж болно. Театрын өнгөнүүд нь
`--primary`, `--secondary-container`, `--tertiary` хувьсагчдаас (эсвэл hex
fallback-аас) уншигддаг тул хүлээн авагч хуудас өөрийн palette-ээ энэ гурван
хувьсагчаар зарлаад light/dark хоёр горимд өөр утга өгч болно.
